Let's Map It!

Enumerable methods are an important part of the Ruby language. They are powerful and can make your life a lot easier by using them. In this post, I will briefly talk about the Enumerable#map. It is a common misconception that the each method is similar to the map method. However, they are not the same. The map method is one of the most powerful enumerables in Ruby. The main difference between each and map is that map returns a completely new object, whereas each does not. Let’s look at how we can use map.

According to the Ruby docs, the Enumerable#map “Returns a new array with the results of running block once for every element in enum.” What does this all mean? When you call map on an array, it will return a new array with whatever you told it to do! Each element will be run through the code block. Let’s see this in practice:

=> numbers = [1, 2, 3, 4, 5]

>> new_numbers = numbers.map { |x| x + 1}

=> [2, 3, 4, 5, 6]

>> new_numbers

=> [2, 3, 4, 5, 6]

As you can see, map returns an entirely new array with the code it ran. This is important to distinguish. If we called each on the numbers variable and assigned it to the new_numbers variable, calling new_numbers would return the original array of numbers. map gives you an entirely new array.

I hope this gives you a basic understanding of what map does. It is an important method in Ruby and is essential in manipulating data. Practice with it in IRB and have fun with it!
