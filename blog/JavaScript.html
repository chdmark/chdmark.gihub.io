JavaScript Objects vs. Ruby Objects

This week we switched gears and dived into JavaScript. While many of the fundamental concepts are the same, the syntax is different and could be frustrating at times. In this post, I will explain the similarities and differences between Ruby hashes and JavaScript objects.

Let’s observe how you would create an object in JavaScript. The first thing to notice is that objects in JavaScript are considered variables as well. Here’s how you create a JavaScript object:

var hero = {

name: “Chris"

class: “Hunter”

power: “Death-strike”

}

You create a JavaScript object by typing the keyword var and using brackets. The way you create a Ruby hash is very similar:

hero = {

name: “Chris"

class: “Hunter”

power: “Death-strike”

}

Notice that in Ruby you don’t need use the word var. Accessing these values are a bit different as well. You can access the class key in JavaScript like this:

console.log(hero.class)

=> Hunter

You use the dot notation to access the key you want and use console.log to print it to the console or terminal. Accessing a hash in Ruby is a bit different:

print hero[:class]

=> Hunter

As you can see, Ruby uses bracket notation to access the values of its keys. There are subtle differences in Ruby hashes and JavaScript Objects. The way you create and access a hash or object is very similar. I wouldn’t say an object or hash is “better”, they are just a bit different in the way they are used.
